keyboard-only

使用 tab 游移和其它按鍵互動的使用者

互動區域使用超連結與buttom

  • 盡量避免另開新視窗
  • tabindex=0,-1
  • accesskey
  • title
  • disabled
  • 文字要明確說明用途且簡短

HTML5

  • 用 h1~h6 編排內容且不得誤用和順序錯誤
  • 使用者會關閉所有圖片載入，一定要有alt且文字要明確說明，重要內容要有上下文
  • 無障礙表單與表格
  • video tag

顏色對比度、避免閃爍與快速動態

沒有說明切確的數字

ARIA、role

類似 microdata 這類的結構化資料標記

小心使用 JavaScript

  • 有些動態內容顯示可能有問題
  • 內容不要像某些廣告一樣突然出現(快速動態)
  • 小心使用 onMouseOver +onFocus、 onMouseOut + onBlur、 onMouseDown + onKeyDown、 onMouseUp + onKeyUp
  • 切確定義主動事件，而不是用 onFocus

CSS focus

  • 重要內容不要用css產生，有些使用者的瀏覽器能自定義樣式
  • grid
  • 必需設計符合色盲色弱的互動
